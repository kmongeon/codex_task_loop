---
name: task-loop
description: Execute bounded Codex tasks using task packets, validation gates, evidence logs, and structured decisions.
---

Use this skill when a Codex task should be handled through a repeatable lifecycle.

Procedure:
1. Read the task packet.
2. Identify objective, allowed paths, blocked paths, deliverables, and acceptance criteria.
3. Make only the smallest scoped change.
4. Let the external loop run validation and artifact checks.
5. Do not mark work accepted unless the evidence satisfies the task packet.
